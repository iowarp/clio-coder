PR Documentation Update Checklist

1. Identify Source Owner

Trace the TS files backing the behavior under src/cli/, src/interactive/, or specific domain scopes.

2. Check Public CLI Help Changes

Verify that all command lines and args mentioned in docs match output from node dist/cli/index.js --help.

3. Update the Mapped Guide

Edit the corresponding docs/*.md file in the same PR containing code improvements.

4. Propagate Core System Changes to README

If modifications affect safety bounds, session ceilings, credentials, or dispatch worker routing, keep user-facing README.md in sync.

5. Execute Python Link Check Script

Execute the local Python script to verify relative links across all files prior to pushing commits: python3 tools/check_links.py.
