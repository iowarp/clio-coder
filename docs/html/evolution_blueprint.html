Clio Coder Recommended Change Workflow

1. Capture the Current Git Commit SHA

Run git rev-parse HEAD to reference the base state of the codebase before any harness modification.

2. Snapshot Component Integrity

Save a component snapshot of key runtime schemas and prompt envelopes before making edits: clio components snapshot --out before.json.

3. Draft the Change Manifest File

Create change-manifest.json containing the targets, predicted regression classes, validation strategies, and authority declarations.

4. Execute Code Improvements & Validate

Write minimal modifications to targets/tools. Execute validation tests and generate test reports or receipts under <dataDir>/evals/ or <dataDir>/evidence/.

5. Validate Manifest Conformity

Run clio evolve manifest validate change-manifest.json to ensure zero schema drift, and check that high-authority paths document regression hypotheses.

6. Promote Change or Rollback

If tests pass and manifest validates, merge the changes. If not, follow the declared rollbackPlan to clean the branch back to the baseline SHA.
