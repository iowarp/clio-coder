Clio Coder Change Manifest Specifications

Change manifests are JSON structures generated or validated using the clio evolve manifest command to support reviewable and falsifiable harness changes.

Change Manifest Schema

Every change must satisfy specific verification keys:

Field Path Type Constraint
iterationId string Required. Only exploratory-1 may omit linked evidence references.
authorityLevel enum Required. Must be one of the nine allowed authority channels.
predictedRegressions string array Required. High-authority changes **must** list at least one regression.
rollbackPlan string Required non-empty string.

Authority levels

Allowed levels include: prompt, tool-description, tool-implementation, middleware, memory, runtime, safety, schema, cli.

High-authority levels that require explicit regression declarations include: tool-implementation, middleware, runtime, safety, schema, cli.
