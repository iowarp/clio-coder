Clio Coder Configuration Specification

Interactive Spec Available: Use the Interactive Config Studio tab above to validate YAML schemas, resolve targets, and generate CLI configurations.

Clio Coder is target-first: chat and fleet dispatch resolve through configured targets in settings.yaml, not through provider-specific ad hoc flags. Chat, print, and fleet dispatch targets are all HTTP/native/pi-ai-backed runtimes.

Source of truth: src/core/defaults.ts, src/core/config.ts, src/domains/providers/**, src/cli/configure.ts, src/cli/targets.ts, and src/cli/auth.ts.

State Locations

Environment Variable Effect
CLIO_HOME Single-tree override: config at $CLIO_HOME/config, data at $CLIO_HOME/data, state at $CLIO_HOME/state, and cache at $CLIO_HOME/cache.
CLIO_CONFIG_DIR Overrides config directory only. Default config directory is platform-specific (e.g. ~/.config/clio on Linux).
CLIO_DATA_DIR Overrides data directory only. Contains durable artifacts such as memory, evidence, and evals.
CLIO_STATE_DIR Overrides state directory only. Contains sessions, audit logs, receipts, runs.json, recent models, install metadata, interviews, and scratch output.
CLIO_CACHE_DIR Overrides cache directory only.

Autonomy Dial

Clio v0.2.6 uses the autonomy dial for immediate execution, approvals, and auto-denials. clio doctor --fix can repair known legacy settings into this shape and backs up the original file first.

  • read-only: Clio inspects and answers. Write/execute tools are auto-denied.
  • suggest: Every mutating action parks for operator confirmation.
  • auto-edit: Workspace writes and recognized commands execute; unrecognized bash prompts.
  • full-auto: Bash commands run without prompting. The safety net remains active.

First-Run Target Setup

Clio works on checked-out code repositories. First register a target and set it as orchestrator:

# Register local Ollama target
clio configure \
  --id local-ollama \
  --runtime ollama-native \
  --url http://localhost:11434 \
  --model qwen2.5-coder:14b \
  --set-orchestrator \
  --set-fleet-default

# Probe registered targets and models
clio targets --probe
clio models --target local-ollama
clio

Live Routing vs Saved Defaults

Routing properties (orchestrator.*, workers.default.*) in settings.yaml are defaults that seed a TUI session on launch. Once running, Alt+L or /model modifications only apply to the current active terminal session and are serialized back for next-session setups without affecting other concurrent running TUI instances.

Writes from multiple processes are synchronized under an advisory lock file (settings.yaml.lock) and saved atomically.
