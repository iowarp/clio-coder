Prompt Envelopes & Tools Spec

Technical architecture specification, compiler calculator, and registry simulator for Clio Coder's secure LLM delivery.
Clio Coder v0.2.3

1. Provider Tool Schema Delivery

Unlike legacy prompt-based wrappers, Clio converts TypeBox schemas into formal provider tool schemas. The model calls tools natively through the LLM API's structural JSON envelope rather than parsing text logs.

2. Prompt Wrapping Envelope & Cache Decision Flow

Clio splits prompts into three tiers (Static, Session, Dynamic) and generates distinct hashes. When supported by targets (e.g. llamacpp or cloud cache), Clio leverages these hashes to skip resending the session shell prompt if it has not changed.

3. Tool Invocation & Result Shaping Lifecycle

Every tool invocation goes through the registry checks, classifying arguments, detecting loop locks, and evaluating security policies. Upon execution, outputs are bounded to respect context pressure through safe UTF-8 truncation and suffix injection.
